# Changelog

## 2026-07-05 Ver1.3

- `chamini6_god_mode` を独立した研究用予測エンジンとして追加し、ロト6・ロト7画面にChamini6 God Modeの統合候補を表示できるようにしました。
- 既存16分析エンジン、AI改善重み、人と被りにくい期待値最大化モデル、セット球分析をChamini6の補助スコアとして統合しました。
- セット球データがない場合でもアプリが停止しないよう、列検出型のセット球分析とフォールバック表示を追加しました。
- モデル成績ランキングの対象にChamini6 God Modeを追加し、保存後に継続評価できるようにしました。
- 保守画面へ読み取り専用のCSV安全診断を追加し、不足列・重複予想ID・検証キー・モデル貢献度ファイル状態を確認できるようにしました。

## 2026-07-04

- Ver1.1としてロト7検証CSVの列不足を補正し、`検証キー` と候補番号、抽せん日、一致数字列を保持できるようにしました。
- ロト7画面の検証重複判定を、予想ID単独ではなく `予想ID + 使用モデル + 候補番号` 相当の検証キーで扱うようにしました。
- ロト7モデル貢献度と研究サイクルを共通CSV列で再生成できる保守経路を確認し、保守画面の表記を Ver1.1 に更新しました。
- `loto_lab/data/backups/` をGit管理外にし、CSV補正前バックアップをローカル保存できるようにしました。

## 2026-07-04 Ver1.2

- 3口予想に Pattern A「本命型」、Pattern B「バランス型」、Pattern C「チャレンジ型」の役割メタ情報を追加しました。
- A/B は最大3個、A/C は最大2個を目安に重複を抑え、共通コア数字1〜2個を許可する後処理を追加しました。
- 補助モデル `anti_popular_expected_value`（人と被りにくい期待値最大化モデル）を追加し、Pattern C で優先採用するようにしました。
- 補助モデルでは、前回当選番号との1個以上の重複、31超え数字の採用、3連続以上の除外、過度な前回重複除外を確認します。
- 予測保存時にPatternごとのモデル名を優先し、補助モデル由来の買い目を検証対象として追跡できるようにしました。
- Chamini6 God Mode を独立エンジンとして接続できる準備レジストリを追加しました。

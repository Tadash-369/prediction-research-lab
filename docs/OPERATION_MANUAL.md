# 通常運用手順書

この手順書は、開発者でなくても毎回同じ流れで「予想 → 結果 → 検証 → 改善」を回すためのものです。

## 1. アプリ起動方法

Python環境で依存ライブラリを入れます。

```bash
pip install -r requirements.txt
```

研究所トップ画面を起動します。

```bash
streamlit run analysis_research_lab.py
```

Windowsでは、以下のバッチファイルから起動できます。

```bat
start_prediction_research_lab.bat
```

個別画面を直接起動する場合は、以下を使います。

```bash
streamlit run loto6_streamlit_app.py
streamlit run loto7_streamlit_app.py
```

## 2. ロト6の予想作成手順

1. アプリを起動します。
2. 研究所トップ画面で「ロト6」を開きます。
3. ロト6専用画面を使う場合は `loto6_streamlit_app.py` を起動します。
4. 当選履歴 `loto6.csv` が読み込まれていることを確認します。
5. 予想生成エリアで次回候補を確認します。
6. 表示された予想番号と理由を確認します。
7. 予想は `predictions.csv` に保存されます。

## 3. ロト6の結果登録手順

1. 抽せん後、ロト6画面を開きます。
2. 「新しい当選結果を登録」欄に開催回、抽せん日、本数字6個、ボーナス数字、球セットを入力します。
3. 本数字とボーナス数字が重複していないことを確認します。
4. 登録ボタンを押します。
5. 結果は `results.csv` に保存されます。

## 4. ロト6の検証手順

1. 結果登録後、検証レポート更新を実行します。
2. 予想履歴と公式結果が照合されます。
3. 検証結果は `verification_reports.csv` に保存されます。
4. モデル貢献度は `loto6_model_contributions.csv` に保存されます。
5. 研究サイクルは `loto6_research_cycles.csv` に保存されます。
6. 本数字一致数、的中率、勝率、期待値、等級判定、失敗要因、改善案、次回の仮説を確認します。

## 5. ロト7の予想作成手順

3. ロト7専用画面を使う場合は `loto7_streamlit_app.py` を起動します。

## 6. ロト7の結果登録手順

4. 登録ボタンを押します。
5. 結果は `loto7_results.csv` に保存されます。

## 8. モデル別成績の見方

モデル別成績では、各分析モデルの過去検証結果を比較します。

主に見る項目は以下です。

- 平均一致数
- 的中率
- 勝率
- 期待値
- 3個以上一致率
- 直近成績

短期の結果だけで判断せず、複数回分の検証結果を見て判断します。

## 9. モデル貢献度の見方

モデル貢献度では、的中した数字がどの分析モデルから選ばれていたかを確認します。

見るポイントは以下です。

- 的中数字に関係したモデル
- 複数モデルで一致していた数字
- 寄与スコアが高いモデル
- 外れた数字に多く関係したモデル

次回予想では、貢献度が高いモデルを重視します。

## 10. AI改善レポートの見方

AI改善レポートでは、検証結果をもとに次回へ向けた改善点を確認します。

確認する項目は以下です。

- 的中要因
- 外れ要因
- 改善案
- 次回仮説

この内容を見て、次回の予想モデルや数字の選び方を調整します。

## 12. 運用時の注意点

- このアプリは当選を保証するものではありません。
- 予想は必ず結果登録前に保存します。
- 結果登録後は必ず検証を実行します。
- ロト6とロト7のCSVを混ぜないようにします。
- CSVを直接編集する場合は、列名を変更しないでください。
- 日本語カラム名を壊さないため、CSVは `utf-8-sig` で保存します。
- 予想、結果、検証、改善を1セットとして保存し続けることが重要です。

## 13. 異常時の確認方法

アプリが起動しない場合は、以下を確認します。

1. 依存ライブラリが入っているか確認します。

```bash
pip install -r requirements.txt
```

2. Streamlitで起動できるか確認します。

```bash
streamlit run analysis_research_lab.py
```

3. CSVファイルが壊れていないか確認します。

確認するポイントは以下です。

- ファイルが存在するか
- 文字化けしていないか
- 列名が変わっていないか
- ロト6とロト7のCSVを入れ違えていないか
- 数字が全角や不要な記号を含んでいないか

4. 検証結果が出ない場合は、以下を確認します。

- 予想履歴が保存されているか
- 同じ開催回の公式結果が登録されているか
- 開催回の数字が一致しているか
- 本数字とボーナス数字が重複していないか

5. それでも直らない場合は、直前に更新したCSVをバックアップから戻し、再度アプリを起動します。
